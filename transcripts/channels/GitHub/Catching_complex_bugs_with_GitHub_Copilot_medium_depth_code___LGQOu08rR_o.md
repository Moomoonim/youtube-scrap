## 스크립트

GitHub Copilot의 코드 리뷰 기능이 더욱 향상되었습니다. 이제 공개 미리보기로 중간 수준 리뷰 기능이 도입되었습니다 . 이러한 검토를 통해 풀 리퀘스트는 복잡한 논리, 보안에 민감한 코드 및 서비스 간 변경 사항에 대한 심층 분석을 위해 더 높은 수준의 추론 모델로 전달됩니다. 따라서 이는 사용자가 가장 많이 의존하고 가장 철저한 검토가 필요한 저장소 및 서비스에 적합합니다. 이 기능을 활성화하려면 저장소의 설정 패널로 이동하세요. Copilot을 선택하세요. 코드 리뷰로 이동하세요. 그리고 매체를 선택하세요. 조직 관리자는 자체적으로 검토 수준을 선택하지 않은 모든 저장소에 대한 코드 검토의 기본 검토 수준을 설정할 수도 있습니다 . 이 일은 당분간 비밀로 해두자. 이 저장소는 제 서비스에 중요한 저장소이므로 , 이 저장소에 대해 중간 수준의 리뷰 기능을 활성화하고 싶습니다. 이제 코드 리뷰를 요청하면 GitHub Copilot이 최신 프론티어 모델을 사용하여 풀 리퀘스트를 분석하고, 더 심층적인 피드백을 제공하며, 놓칠 수 있는 미묘한 문제까지 잡아냅니다. 참고로, 중간 수준은 일반적으로 낮은 수준보다 더 많은 AI 크레딧을 소모하는데, 이는 코드에 대한 보다 심층적인 분석을 제공하기 위함입니다. 이 경우, 저는 정수 계산에서 연산 순서상 나눗셈 전에 곱셈을 해야 했기 때문에 회계 오류가 발생했습니다. 그렇지 않았다면 예산 사용률이 100%가 될 때까지 0%로 표시되었을 것입니다 . 이미 일반적인 문제 해결률이 향상되어 70% 이상의 해결률을 보이고 있습니다. GitHub Copilot 코드 리뷰에서 중간 수준 리뷰 기능이 현재 공개 미리보기로 제공되고 있습니다.
