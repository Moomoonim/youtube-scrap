## 스크립트

이 데모에서는 스토리지 마이그레이션 에이전트가 Azure 포털에서 단일 대화를 통해 온프레미스 SMB 파일 공유를 Azure 파일로 직접 마이그레이션하는 전체 과정을 안내합니다 . 시작하기 전에 준비해야 할 네 가지가 있습니다. 먼저, 마이그레이션 에이전트가 실행되는 Azure 마이그레이션 프로젝트가 포함된 Azure 구독이 필요합니다. 둘째, 마이그레이션할 SMB 공유 경로와 해당 자격 증명을 입력하세요. 셋째, 마이그레이션 어플라이언스를 실행할 호스트가 필요합니다. 네 번째는 해당 기기의 네트워크 연결성입니다. 상담원은 채팅을 통해 키 볼트, 스토리지 계정, 스토리지 이동 도구와 같은 리소스를 생성하며, 사용자의 승인 없이는 아무것도 실행되지 않습니다. 시작하겠습니다. 에이전트는 먼저 두 가지 진입 경로를 제시합니다. 첫 번째 옵션은 환경에서 사용 가능한 파일 공유를 식별하기 위해 자동 검색을 수행합니다 . 두 번째 옵션을 사용하면 관리자가 알려진 공유 경로를 직접 제공할 수 있습니다. 이 안내에서는 공유 경로가 제공됩니다. 소스가 확인되면 에이전트는 마이그레이션에 필요한 SMB 자격 증명을 안전하게 저장하기 위해 Azure 키 자격 증명 모음을 준비합니다. 자격 증명은 대화에 절대 언급되지 않습니다. 제안된 구성은 검토 및 승인을 위해 제출되었습니다. 설계상 에이전트는 자격 증명을 직접 처리하지 않습니다. 관리자는 사용자 이름과 암호를 키 볼트의 비밀 키로 추가하고, 해당 비밀 키 이름을 에이전트에 다시 제공하여 작업을 계속 진행합니다. 다음으로 에이전트는 대상 스토리지 계정과 파일 공유를 제안합니다. 요청하신 지역을 이용할 수 없는 경우, 가장 가까운 지원 지역이 자동으로 선택되며 그 이유가 명확하게 표시됩니다. 승인이 완료되면 스토리지 이동 리소스와 마이그레이션 프로젝트가 프로비저닝됩니다. 스토리지 이동 서비스는 실제 데이터 전송을 수행하기 위해 온프레미스 장비가 필요합니다 . 관리자는 어플라이언스를 배포하고 등록한 다음, 어플라이언스가 온라인 상태가 되었음을 에이전트에 알립니다 . 그런 다음 담당자는 최종 검토를 위해 전체 마이그레이션 작업을 제시합니다. 승인이 완료되면 Storage Mover를 통해 마이그레이션 작업이 시작됩니다. 작업이 완료되면 에이전트는 스캔된 바이트 수, 전송된 바이트 수, 성공한 항목 및 실패한 항목을 포함한 전체 요약을 반환합니다. 마이그레이션이 이제 완료되었으며, 단일 안내 프로세스를 통해 완벽하게 진행되었습니다.
