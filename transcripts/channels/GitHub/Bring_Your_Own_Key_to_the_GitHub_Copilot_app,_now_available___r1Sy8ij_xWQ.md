## 스크립트

이제 GitHub Copilot 앱에서 '자신의 키를 직접 가져오세요' 기능을 사용할 수 있습니다 . 설정 방법을 보여드리겠습니다 . 왼쪽 하단의 설정을 클릭하고, 모델 공급자를 클릭한 다음, 여기에서 Anthropic, Azure OpenAI, 사용자 지정 엔드포인트, Foundry Local 및 Ollama를 통한 로컬 모델까지 다양한 옵션으로 공급자 추가를 클릭할 수 있습니다 . 이것들 중 하나를 클릭하면 편집할 수 있으며, 표시 이름, 기본 URL, 사용할 와이어 API, API 키, 그리고 필요한 경우 사용자 지정 헤더를 추가하는 것만큼 간단하다는 것을 알 수 있습니다. '공급 업체 추가'를 클릭하면 Ollama를 선택할 수 있습니다 . 이미 설정과 설치가 완료되어 있으므로, 공급자 추가 버튼을 클릭하기만 하면 됩니다. ' 카탈로그 찾아보기'를 클릭하면 Ollama를 통해 설치한 모든 모델을 볼 수 있습니다. 그리고 여기 있는 오르니스(Ornith)를 클릭하면 끝입니다 . 이 창을 닫으면 이제 모델 선택기를 클릭하여 제가 만든 모든 사용자 지정 공급자 모델을 사용할 수 있는 것을 확인할 수 있습니다. GitHub Copilot 앱은 모든 GitHub Copilot 구독과 함께 사용하거나, 구독 없이도 사용할 수 있습니다. 따라서 GitHub에서 호스팅되는 모델, 자체 키(BYOK)를 통해 생성한 모델, 또는 이 둘 모두를 한 곳에서 사용할 수 있습니다. 이렇게 하면 모든 생성형 작업에 하나의 앱만 사용하면 됩니다. 이제 실제 워크플로에서 사용자 지정 공급자 모델 중 하나를 사용해 보겠습니다. 저는 ' 내 작업'을 클릭할 거예요. 그러면 선택한 저장소에 있는 모든 미해결 이슈와 PR 목록을 한눈에 볼 수 있죠. 저는 96번 이슈를 작업하고 싶어요. 그럼 96번 이슈를 클릭할게요. 오른쪽 상단에서 ' 새 세션'을 클릭하여 이 이슈를 컨텍스트로 하는 작업 트리와 세션을 생성하겠습니다 . 이 문제를 함께 계획해 봅시다. 이 작업을 수행하는 데 Cprofile 및 Pi Instrument 외에 사용할 수 있는 추가 도구도 조사해 보시기 바랍니다. 그리고 저는 먼저 계획을 세우고 싶으니 , 이 창을 계획 모드로 옮기겠습니다. 좋습니다, 그럼 보내드리겠습니다. 그리고 GitHub Copilot이 이 작업을 수행하고 있는 것을 확인할 수 있습니다 . 물론 제 작품을 클릭해서 볼 수도 있습니다. 다시 한번, 이 중 아무거나 하나 클릭해 보겠습니다. 이걸 클릭하고 세션을 연 다음 GitHub Copilot 모델 중 하나를 선택하겠습니다 .
