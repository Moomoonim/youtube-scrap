## 스크립트

이 튜토리얼에서는 Oracle Fusion Applications 구독을 활성화하고 서비스를 호스팅하는 데 필요한 Oracle Cloud 계정을 생성하는 방법을 알아봅니다. 계정 관리자로서 Oracle Fusion Applications를 프로비저닝하기 전에 구독을 활성화하고 Oracle Cloud 계정과 연결해야 합니다 . 이 시나리오에서 저희 조직은 Oracle Fusion ERP Cloud 서비스를 구매했으며, 구현 계획에 따라 서비스를 호스팅하기 위한 새로운 Oracle Cloud 계정이 필요합니다. 오라클에서 구독 주문을 처리한 후 , 구독 활성화 정보가 포함된 환영 이메일을 찾겠습니다. 환영 이메일은 판매 주문서에 기재된 담당자에게 발송되며, 오라클 클라우드 구독 관리를 담당하는 관리자에게 전달할 수 있습니다 . 진행하기 전에 클라우드 서비스 구독이 주문하신 Fusion Applications 서비스와 일치하는지 확인하겠습니다. 이 예시에서 구독 대상은 Oracle Fusion ERP 클라우드 서비스입니다. 이 단계에서는 기존 Oracle Cloud 계정에 서비스를 추가 하거나 새 계정을 만들 수 있습니다. 이 구현에는 별도의 환경이 필요하므로 새 클라우드 계정 생성을 선택하겠습니다 . 새 클라우드 계정 정보 페이지가 열리면 관리자의 이름과 성을 입력하겠습니다. 이메일 주소는 자동으로 입력됩니다. 다음으로, 요구되는 비밀번호 조건을 충족하는 비밀번호를 생성하고 확인하겠습니다. 환영 이메일에 명시된 데이터 센터 지역을 기준으로 홈 지역을 선택하겠습니다. 활성화 후에는 홈 지역을 변경할 수 없습니다. 클라우드 계정 유형으로는 표준 테넌트를 선택하겠습니다. 페이지에 표시된 기준에 맞는 클라우드 계정 이름을 입력하겠습니다. 계정 이름은 로그인 URL의 일부가 되며 나중에 변경할 수 없습니다 . 필요한 정보를 모두 입력했으면 클라우드 계정 생성을 선택하겠습니다 . 활성화 요청이 제출되었다는 확인 메시지가 표시됩니다. 활성화 과정은 완료하는 데 최대 1시간이 소요될 수 있습니다. 활성화가 완료되면 오라클에서 시작 방법에 대한 안내가 담긴 이메일을 받게 됩니다. 이제 구독을 활성화 하고 클라우드 계정을 생성했으므로 로그인하여 활성화를 확인하고 Oracle Fusion ERP 클라우드 서비스 프로비저닝을 시작할 수 있습니다. 아래 링크에서 Oracle Fusion Applications 배포 관련 추가 자료 및 튜토리얼을 찾아보세요.
