## 스크립트

이 데모에서는 대규모 언어 모델을 위한 ETO 파이프라인 플랫폼인 DataVerse의 기본 사항을 보여드리겠습니다. 오픈 소스 GitHub 페이지에 있는 예제를 통해 DataVerse 사용 방법을 보여드리겠습니다. 예제를 살펴보기 전에 시스템 아키텍처를 간략하게 살펴보겠습니다. DataVerse ETO 파이프라인은 인터페이스이며 사용자는 구성과 상호 작용하게 됩니다. 기본적으로 사용자가 알아야 할 사항은 다음과 같습니다. 자, 이제 파이프라인이 구축 되고 실행됩니다. 아주 간단합니다. 이제 사용자 친화적인 인터페이스를 통해 DataVerse가 어떻게 작동하는지 예제를 통해 보여드리겠습니다. 먼저 사용 방법을 알아보겠습니다. 여기처럼 구성을 준비해야 합니다. 구성은 Spark 설정과 사용할 데이터 프로세서, 두 가지로 구성됩니다. 간단히 구성 후 ETL 파이프라인을 인스턴스화하고 구성을 적용하면 됩니다. 결과로 데이터 세트가 반환되는 것을 여기에서 볼 수 있습니다. 다음으로 사용자 지정 데이터 프로세서를 추가하는 방법을 살펴보겠습니다. 예를 들어 허깅 페이스 데이터 세트를 사용해 보겠습니다. DataVerse는 허깅 페이스 데이터 세트를 지원하며, 여기에 간단히 입력하면 됩니다. 이처럼 아주 간단한 AOSC 챌린지입니다. 데이터 세트를 확인하다가 ' 선택 열'과 같은 것을 발견했는데, 이를 제거하고 싶다고 가정해 보겠습니다. 어떻게 하면 될까요? 아주 간단합니다. 사용자 정의 데이터 프로세서를 정의하고 해당 열을 제거한 다음, 데코레이터 레지스터를 추가하면 됩니다. 이제 구성에 추가하고 이전처럼 ETO 파이프라인을 실행하면 됩니다. 보시다시피 더 이상 선택 열이 없습니다. 마지막으로, 확장을 위해 CMR에서 어떻게 실행할까요? 아주 간단합니다. 이전과 마찬가지로 EMR 옵션을 true로 설정하면 됩니다.
