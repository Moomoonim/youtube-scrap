## 스크립트

[음악] 안녕하세요 여러분, 환영합니다. 이 비디오에서는 Azure Migrate를 사용하여 애플리케이션을 Azure로 마이그레이션하거나 현대화하는 계획을 세우는 방법을 살펴보겠습니다. 먼저 계획 단계를 살펴보겠습니다. Azure Migrate를 사용하면 ASP.NET 웹 애플리케이션, SQL Server, Windows Server, Linux Server 등과 같은 다양한 워크플로를 검색하고 이러한 워크로드를 적절하게 그룹화하여 애플리케이션을 정의할 수 있습니다. 특히 애플리케이션 마이그레이션 또는 현대화 측면에서 볼 때 , 새로운 애플리케이션 평가 기능은 애플리케이션을 구성하는 워크로드의 마이그레이션 또는 현대화 준비 상태와 장애 요인을 철저히 파악하고, 특정 마이그레이션 의도 또는 선호도에 따라 리호스팅 또는 리플랫폼과 같은 이상적인 마이그레이션 전략을 제시하며, 대상 Azure 서비스, 적절한 SKU, 예상 비용 및 이러한 권장 사항의 근거를 권장하고, 권장 접근 방식을 사용하여 애플리케이션을 마이그레이션 또는 현대화하는 데 필요한 단계를 설명하는 마이그레이션 지침을 제공하도록 설계되었습니다. 다계층 애플리케이션을 Azure로 마이그레이션하기 위한 평가 및 효과적인 계획을 수립하기 전에, 애플리케이션을 구성하는 워크로드의 성공적인 검색에 필요한 모든 전제 조건을 충족하고 작업 센터 에 표시된 모든 검색 문제를 해결해야 합니다. 필수 조건에 대한 자세한 내용은 aka.ms/azuremigrate/discovery를 참조하고 , 워크로드 검색에 필요한 최소 권한 자격 증명을 생성하는 방법에 대한 자세한 내용은 aka.ms/azuremigrate/discoveryprivileges를 참조하세요 . 또한, 단기간 내에 현대화 또는 마이그레이션을 진행하려는 의도를 바탕으로, 마이그레이션 또는 현대화에 대한 투자 수익률(ROI)을 파악하기 위한 사업 타당성 분석을 수행하는 것이 좋습니다 . 탐색 과정이 성공적으로 완료되면 개요에서 ' 평가 생성'을 클릭하세요. 이번 데모에서는 Airsonic과 Parts Unlimited라는 두 가지 애플리케이션을 평가해 보겠습니다. 평가 생성 화면에서 알아보기 쉬운 평가 이름을 지정 하고 평가 범위에 애플리케이션을 추가하세요 . AirSonic은 PGSQL 데이터베이스를 사용하는 Java 웹 애플리케이션이고, Parts Unlimited는 SQL DB 백엔드를 사용하는 .NET 웹 애플리케이션입니다 . 또한, 평가를 생성할 때 애플리케이션을 배포할 지역 , 애플리케이션 환경 , 원하는 요금제 또는 플랜( 종량제 또는 기업 계약 등)을 선택할 수 있습니다. 뿐만 아니라 웹 계층 에 선호하는 PaaS 서비스와 같은 워크로드별 고급 설정도 수정할 수 있습니다 . 평가 범위에 포함된 작업량 유형에 따라 작업량별 설정이 표시됩니다. 원하는 설정을 선택했으므로 이제 평가를 생성하겠습니다. 신청서 평가에서는 신청자의 이민 선호도에 따라 권장 사항을 제공합니다. 이 경우, 현대화 방식이 상당한 비용 절감을 가져오기 때문에 저희는 현대화 방식을 선호합니다. 애플리케이션 평가를 통해 PaaS 선호, PaaS 전용, Azure VM을 사용한 리프트 앤 시프트, AVS를 사용한 마이그레이션 등 가능한 모든 마이그레이션 경로를 제공해 드립니다. Azure 준비 상태, 총 소유 비용, 예상 절감액 등 주요 특징을 기준으로 여러 경로를 비교할 수 있습니다. 여기서는 애플리케이션을 구성하는 워크로드를 쉽게 재플랫폼화할 수 있다는 점이 분명하게 드러납니다 . Azure Migrate는 애플리케이션을 구성하는 워크로드에 가장 적합한 대상 서비스를 자동으로 권장합니다 . AirSonic 애플리케이션의 경우 , 평가 결과 PGSQL DB는 Azure Flexible Server for PostgreSQL로, 웹 앱은 앱 서비스로 이전하는 것이 권장됩니다. 더 자세히 살펴보면 각 워크로드에 대한 SKU 및 비용 세부 정보와 권장 마이그레이션 도구를 확인할 수 있습니다. 마찬가지로 Parts Unlimited의 경우에도 평가 결과 데이터 계층은 Azure SQL Managed Instance로, 웹 계층은 App Service로 플랫폼을 변경하는 것이 좋다는 권고를 받았습니다. 이와 더불어 Azure Migrate를 사용하면 애플리케이션 현대화에 필요한 코드 변경 사항을 시각화할 수도 있습니다 . 코드 분석 정보는 CAST Highlights 또는 GitHub Copilot 코드 평가에서 가져올 수 있습니다. 우선 CAST 하이라이트에서 코드 관련 인사이트를 살펴보겠습니다 . '코드 인사이트 추가'를 클릭하고 CAST 하이라이트 대시보드에서 가져오거나 CAST 자격 증명을 사용하여 토큰으로 새 연결을 생성하기만 하면 됩니다. 연결이 설정되면 애플리케이션 이름을 매핑 하고 코드 변경 사항을 가져옵니다. 이제 Parts Unlimited의 현대화를 위해 필요한 코드 변경 사항을 확인할 수 있습니다 . 또한 이를 통해 평가 품질이 향상되었음을 알 수 있습니다. 애플리케이션에 코드 변경이 필요했기 때문에 마이그레이션 전략이 리팩토링으로 변경되었습니다. 하지만 언제든지 기본 설정을 변경할 수 있으며, Azure Migrate는 애플리케이션을 구동하는 각 워크로드에 적합한 전략을 권장합니다 . 시간이 부족하다면 환경 설정을 최신 버전으로 업데이트할 수 있습니다 . 보시는 바와 같이 Azure Migrate는 애플리케이션을 구동하는 워크로드에 대해 현대화 전략보다는 리프트 앤 시프트 전략을 권장합니다 . 데이터 센터의 모든 애플리케이션을 포괄하는 애플리케이션 평가를 생성하려면 전체 인벤토리 보기에서 모든 워크로드를 선택 하고 평가 생성을 선택하기만 하면 됩니다 . 계획 단계가 끝나면 실행 단계가 이어집니다. 탐색 및 평가가 완료되고 신뢰도 높은 마이그레이션 계획을 수립했다면 Azure Data Studio, Azure Database Migration Service DMS 및 SQL Server Migration Assistant를 활용하여 애플리케이션의 데이터 계층을 현대화할 수 있습니다 . Azure Migration 컨테이너화 툴킷, App Service 마이그레이션 도우미, Azure Migration 애플리케이션 및 코드 평가 도구를 사용하여 애플리케이션의 웹 계층을 현대화하고 , Azure Migration 서버 마이그레이션 기능을 통해 서버를 Azure 가상 머신으로 마이그레이션할 수 있습니다.
