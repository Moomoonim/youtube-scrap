## 스크립트

AI는 팀이 더 많은 플랫폼에 더 많은 코드를 이전보다 더 빠르게 배포할 수 있도록 지원하지만, 속도는 팀이 배포된 코드를 신뢰할 수 있을 때만 의미가 있습니다. GitHub의 코드 품질 기능이 이제 일반에 공개되었습니다. 오늘은 API에 대한 진단 기능을 소폭 개선하는 작업을 진행하고 있습니다. 변경 사항은 빌드되지만 런타임 에 오류가 발생할 수 있는 미묘한 안정성 문제를 야기합니다 . 형식 문자열은 자리 표시자 0과 2를 참조하지만, 인덱스 0과 1 두 값만 제공됩니다. 이로 인해 런타임 시 서식 예외가 발생할 수 있습니다 . GitHub의 코드 품질 관리 도구는 CodeQL을 사용하여 이를 감지하고 변경된 풀 리퀘스트 라인에 직접 플래그를 지정합니다. 이는 명확한 설명과 구체적인 수정 사항이 있는 신뢰성 오류로 분류됩니다. 개발자는 맥락을 유지하고 , 문제 해결이 작업 단위가 됩니다 . Copilot 자동 수정 기능은 동일한 풀 리퀘스트에서 수정 사항을 제안합니다. 제가 직접 검토하고, 필요하면 조정하고, 바로 커밋할 수 있습니다. 자동 적용이나 자동 병합 기능이 없어 개발자가 모든 것을 제어할 수 있습니다. 규칙 세트는 코드 배포 전에 병합 경계에서 이러한 품질 표준을 적용합니다. 수정 후 코드 품질 검사가 다시 실행되고, 문제가 해결된 것으로 확인되면 풀 리퀘스트를 다음 단계로 진행할 수 있습니다. 풀 리퀘스트 검사를 통해 새로운 문제가 코드베이스에 유입되는 것을 방지합니다. 저장소 수준 보기를 통해 팀은 코드베이스 전반에 걸쳐 기존 안정성 및 유지 관리 작업의 우선순위를 정하고 시간 경과에 따른 진행 상황을 추적할 수 있습니다. 코드 품질은 확정적인 CodeQL 분석 과 AI 기반 분석 결과를 결합하여 제공하며, 각 결과는 명확하게 표시되어 팀에서 결과 도출 방식을 쉽게 파악할 수 있습니다. GitHub의 코드 품질 관리 기능은 풀 리퀘스트부터 조직 차원의 관점까지, 개발자와 엔지니어링 리더에게 소프트웨어 상태에 대한 공통된 시각을 제공합니다 . 코드 품질은 AI 시대 소프트웨어 개발의 핵심적인 강제 요소입니다 . 문제를 더 일찍 발견하고, 상황에 맞게 수정하고 , 병합 전에 적용하세요.
