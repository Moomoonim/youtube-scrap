## 스크립트

[음악] GitHub Copilot 앱은 에이전트 우선 방식으로 GitHub를 처음부터 다시 구축하고 재구상할 수 있는 기회였습니다. 오전 6시, 노트북을 켜고 작동시킨다 . 나는 커피를 끓이러 간다. 그때쯤이면 제 일과는 모두 끝나고, 그 후에 모든 업무 결과를 검토합니다. 여기 보시면 제가 다양한 문제 분류 워크플로를 모두 가지고 있는데, 그중에서 제가 가장 좋아하는 건 오늘 아침 브리핑이에요. 처음부터 제가 확인하고 살펴볼 것들의 목록, 즉 오늘 해야 할 일 다섯 가지를 간략하게 정리한 목록을 가지고 있습니다. 바로 여기서 버그를 발견했어요. 제가 직접 세션을 열어서 해당 PR을 생성한 엔지니어가 온라인에 접속할 때 바로 작업을 시작할 수 있도록 차단 해제를 해두겠습니다. 출시 직후부터 쏟아지는 문의량에 제가 일일이 대응하기가 불가능할 정도입니다. 저는 팀즈, 아웃룩, 슬랙을 사용합니다. 제가 관리해야 할 커뮤니케이션 채널이 너무 많아서 모든 것을 파악하기가 정말 어렵습니다. 하루에 몇 번씩 100개가 넘는 메시지를 읽고 그중에서 정말 중요한 10개를 추려내는 대신 , Work IQ MCP 서버 워크플로우를 실행해서 보고서를 생성하도록 할 수 있습니다. 이미 최우선 순위 항목으로 추려졌습니다. GitHub Copilot 앱을 사용하면 에이전트를 쫓아다니는 대신 에이전트와 파트너십을 맺는 느낌이 듭니다. 이러한 워크플로를 통해 제가 원하는 작업 방식을 설명하고, 불필요한 정보를 걸러내어 제게 정말 필요한 것들만 남길 수 있습니다. GitHub Copilot 앱 덕분에 음악 작업이 다시 즐거워졌습니다.
