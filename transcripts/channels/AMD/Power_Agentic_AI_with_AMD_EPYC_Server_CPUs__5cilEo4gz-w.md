## 스크립트

Agentic AI changes the infrastructure conversation. It's not one workload running in one place. It's a full pipeline of activity, orchestration, retrieval, tools of many kinds, memory, security, and a diverse set of workflow execution. When agentic AI moves into production, what you're trying to optimize for is not how quickly a single agent executes. It's more about how a fleet of agents can get as much work done as possible in a given time. If I could use an analogy, a theme park doesn't optimize for the wait time of a single rider. It optimizes for how many riders it can get through in an hour. So the real scaling metric for agentic AI is concurrency, and not about any single agent doing something faster. AMD has been working with hyperscalers and enterprises for generations now, optimizing for a broad set of workloads. And that optimization helps us handle all of the different agentic tasks. AMD EPYC server CPUs are built for that kind of scale-out enterprise infrastructure today, with the broadly available 5th generation Turin CPUs, and Venice CPUs coming this year to extend that path for the next generation.
