## 스크립트

이 비디오에서는 하나의 Jira 티켓을 사용하여 Copilot을 실행하는 세 가지 방법을 살펴보겠습니다. 이슈를 할당하거나, 댓글에 Copilot을 언급하거나, Atlassian 자동화 기능을 사용하여 작업을 자동으로 인계하는 방법이 있습니다. 이 티켓을 앞으로 계속 사용하겠습니다. Jira에서 Copilot으로 이 작업을 전달하는 방법은 세 가지가 있으므로 하나씩 살펴보겠습니다 . 가장 빠른 방법은 GitHub Copilot을 담당자로 지정하는 것입니다. 티켓을 배정하는 즉시 Copilot이 작동을 시작합니다. 만약 팀에서 담당자 필드를 유지하고 있다면, 댓글에 Copilot을 언급하기만 하면 동일한 결과를 얻을 수 있습니다. Copilot을 Atlassian 자동화 기능에 연결할 수도 있습니다. 이 예시에서는 '개발 준비 완료' 상태로 이동된 모든 티켓이 자동으로 GitHub Copilot에 할당됩니다. 누구도 각 티켓을 수동으로 접수할 필요가 없습니다. Copilot은 변경 작업을 시작하기 전에 한두 가지 질문을 할 수 있습니다. 상담원 상태 표시줄을 확인하여 질문이 있으면 채팅 패널에서 직접 답변해 주세요 . 여기서는 모델을 선택 하고 대상 분기를 확인합니다. 우리가 답장을 보내면 Copilot이 그 내용을 감지하고 계속 진행합니다. 만약 같은 질문에 반복적으로 답하고 있다면, 다음 영상에서 다룰 공간 설정으로 기본 설정을 옮기는 것이 좋다는 신호입니다. Copilot이 작동하는 동안 채팅창을 열어 전체 상황을 실시간으로 확인하세요. Copilot이 저장소를 읽고, 계획을 생성하고 , 파일을 편집하고, 검사를 실행하고, 풀 리퀘스트를 준비하는 실제 활동 로그를 확인할 수 있습니다. 새로 고침할 필요도 없고, Jira를 종료할 필요도 없이 최신 정보를 계속 확인할 수 있습니다. Copilot이 세션을 완료하면 후속 조치를 위한 적절한 위치를 선택하여 워크플로를 계속 진행할 수 있습니다. Jira 작업 항목에 직접 댓글을 달면 Copilot이 새 실행을 시작하고 새 PR을 엽니다. Copilot이 이미 생성한 풀 리퀘스트를 수정하거나 개선하려면 채팅 창에서 대화를 이어가세요 . 새로운 작업을 시작 하거나 별도의 풀 리퀘스트를 생성하려면 Jira 작업 항목에 직접 댓글을 남겨주세요. 풀 리퀘스트가 검토 준비가 완료되면 Copilot은 Jira 작업 항목에 PR 링크가 포함된 댓글을 남깁니다. 그다음 GitHub에서 풀 리퀘스트를 열고 저장소의 변경 사항을 확인할 수 있습니다. 팀원들은 자신들이 일하는 곳에서 업무를 인계받습니다 . 티켓 관리는 Jira, 풀 리퀘스트는 GitHub를 사용하고, 코드 검토 시점에는 검토 준비가 완료된 명확한 체크포인트를 만들어 둡니다 . 그게 바로 매일 반복되는 일상입니다. 티켓을 생성하고, 팀에서 선호하는 방식으로 Copilot을 실행하고, Jira에서 질문에 답변하고, 진행 상황 스트림을 확인하고, 준비가 완료되면 풀 리퀘스트를 검토하세요.
