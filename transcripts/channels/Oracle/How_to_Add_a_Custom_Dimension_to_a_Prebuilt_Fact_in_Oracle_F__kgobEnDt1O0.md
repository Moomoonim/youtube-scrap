## 스크립트

이 튜토리얼에서는 시맨틱 모델 확장 프레임워크를 사용하여 Oracle Fusion Data Intelligence(FDI)의 사전 구축된 팩트에 사용자 지정 차원을 추가하는 방법을 알아봅니다 . 이 시나리오에서 저는 시맨틱 모델 관리자입니다. 저는 시맨틱 모델 확장 기능을 사용하여 SCM 영업 주제 영역의 기존 팩트 테이블에 추가 데이터가 포함된 부서 차원을 추가할 것입니다 . 논리적 스타에 차원을 추가하면 수치적 사실에 추가적인 맥락과 설명적 속성을 제공할 수 있습니다 . 먼저 Autonomous Data Lakehouse에서 사용자 지정 부서 차원을 생성하겠습니다 . 차원은 여러 가지 다른 방식으로 개발될 수 있습니다. 저는 데이터 증강, 특수 ETL 또는 데이터 통합 ​​도구를 사용하거나 수동 또는 프로그래밍 방식으로 SQL을 실행할 수 있습니다. 조인 키 열이 고유한지 확인하는 것이 중요합니다. 그렇지 않으면 통합 문서에 데이터가 중복됩니다. 다음으로, OAX 달러 OAC 스키마에 대한 사용자 지정 차원에 부서 테이블에 대한 시맨틱 모델 액세스 권한을 부여하겠습니다. 이를 통해 FDI의 의미 모델이 새 테이블에 접근할 수 있게 됩니다. 자 , 이제 시맨틱 모델 확장 페이지를 열겠습니다. FDI 홈페이지에서 콘솔로 이동한 다음, 시맨틱 모델 확장으로 이동하겠습니다. 이렇게 하면 확장 프로그램을 게시할 준비가 될 때까지 안전하게 모델을 변경할 수 있는 샌드박스 목록으로 이동합니다 . ' 작업 수행'을 클릭하고, ' 논리적 별 관리'를 클릭한 다음, '논리적 별 편집'을 클릭하겠습니다. 기본 설정을 선택한 다음 판매 주문 주제 영역과 차원을 추가하려는 항목을 선택합니다. 다음 버튼을 클릭하면 논리 스타가 표시됩니다. 논리적 별은 중심에 사실이 있고 주변 차원들과 연결된 차원 모델의 기본적이고 완전한 단위입니다. 객체, 속성, 조인 및 계산을 추가하고 업데이트하여 논리적 스타 구조를 관리할 수 있습니다. 논리적 별 모양에서 차원 추가를 클릭하고 새 차원을 생성합니다. 이전에 생성한 데이터베이스 차원에 스키마를 입력하겠습니다. 다음으로 차원의 이름을 지정하고 속성과 키를 선택하겠습니다. 이 시나리오에서는 부서 차원에 대한 계층적 탐색이나 드릴다운 기능이 제공되지 않습니다 . 그래서 저는 계층 구조를 표시할 필요가 없으므로 '다음'을 클릭하고 ' 주제 영역에 계층 구조 추가' 옵션을 선택 해제하겠습니다 . 제목 영역에는 표시되지 않지만, 최소한 하나의 레벨과 관련 키를 포함한 이름을 제공해야 합니다. 계층 구조의 이름을 사용자 지정 부서 계층 구조로 지정하겠습니다. 시맨틱 모델에 표시하고 싶은 모든 데이터 요소를 이 레벨 또는 세부 정보로 추가하겠습니다. 정렬 순서를 지정할 필요가 없습니다. 준비가 되면 완료를 클릭하여 논리적 스타로 돌아가겠습니다. 다음으로, 새 부서 차원을 기존 팩트에 연결하겠습니다. 해당 항목을 클릭하고 ' 확장 프로그램 관리'를 선택하겠습니다. 열을 추가한 다음 물리적 열을 추가하여 조인 열을 추가합니다. 기존에 정의된 열 이름과의 충돌을 방지하기 위해 필요에 따라 논리 열의 이름을 변경하겠습니다 . 그다음에 완료를 클릭하겠습니다. 물리적 열을 선택할 때, 해당 열이 올바른 테이블에서 가져온 것인지 확인해야 합니다 . ' 사용자 지정만 표시'를 클릭하고 사용자 지정 차원을 미리 만들어진 판매 팩트 테이블 위로 드래그하여 조인 대화 상자를 열겠습니다. ' 조인 조건 추가'를 클릭하고 이전에 팩트 테이블에 추가한 두 개의 키를 기준으로 조인하겠습니다 . 조인 유형을 왼쪽 외부 조인으로, 콘텐츠 수준을 상세 조인으로 설정하겠습니다. 논리적 별표로 돌아가서 해당 항목을 마우스 오른쪽 버튼으로 클릭하여 확장 프로그램을 관리하겠습니다. 이전에 추가한 열을 수정 하고 사용자 지정 차원 수준을 세부 정보로 설정하겠습니다. 이제 의미 모델에서 사용자 지정 차원을 표현할 수 있습니다 . 샌드박스로 이동하겠습니다 . ' 작업 수행'을 클릭하세요. 주제 영역 관리를 선택한 다음 주제 영역을 수정하세요. SCM 판매 주문 주제 영역을 선택하겠습니다. 요소를 추가하려면 다시 다음을 클릭한 다음 새 사용자 지정 요소를 추가하겠습니다. 부서 속성 열을 선택한 다음 적용을 클릭하겠습니다. 필요에 따라 순서를 바꾸거나, 구성을 변경하거나, 이름을 바꾼 다음 완료를 클릭할 수 있습니다. 변경 사항이 준비되면 업데이트된 모델을 적용하고 게시하겠습니다. 이 단계를 통해 우리 조직 전체에 걸쳐 보고 및 분석에 새로운 차원을 도입할 수 있게 됩니다. 샌드박스에서 작업 메뉴를 선택하고 변경 사항 적용을 선택합니다. 활동 탭에서 요청 상태를 확인할 수 있습니다. 의미 모델 구성 요소 탭에서 ' 모델 게시'를 클릭합니다. 내 샌드박스를 선택한 다음 게시를 클릭하세요. 활동 탭에서 게시가 성공적으로 완료되었음을 확인할 수 있습니다 . 마지막으로, 확장된 주제 영역을 사용하여 새 통합 문서를 만들어 모든 것이 올바른지 확인하겠습니다 . 부서 속성을 사용하여 시각화를 생성하고, 올바른 정보가 표시되는지, 그리고 예상대로 팩트 측정값과 조인되는지 확인하겠습니다. 이제 사용자 지정 부서 차원을 기존 팩트에 추가했으므로 우리 조직 사용자는 부서별로 판매 데이터를 분석할 수 있습니다 . 아래 링크에서 이 시리즈의 다른 비디오와 Oracle Fusion Data Intelligence 사용에 대한 추가 자료를 찾아보세요.
