## 스크립트

[음악] 이 비디오에서는 Azure Migrate 어플라이언스를 사용하여 VMware 환경을 검색하는 방법을 배우게 됩니다 . 진행하기 전에 이 비디오 링크를 시청하고 Azure Migrate 프로젝트를 생성했는지 확인하십시오. 어플라이언스를 배포하기 전에 다음 필수 조건이 충족되었는지 확인하십시오. 해당 장비가 모든 네트워킹 요구 사항을 충족하고 필요한 엔드포인트에 연결되는지 확인하십시오. 읽기 전용 및 게스트 운영 권한이 있는 vCenter 계정을 프로비저닝합니다. 이 계정을 통해 어플라이언스는 VMware 서버 의 기본 구성을 검색 하고 성능 데이터를 수집할 수 있습니다. 설치된 소프트웨어, 서버 종속성, SQL, PostgreSQL, MySQL 데이터베이스 및 웹 애플리케이션 검색을 포함한 게스트 및 워크로드 검색을 활성화하려면 Windows 관리자 및 Linux 루트 자격 증명이 필요합니다. 또는 조직의 보안 정책으로 인해 관리자 또는 루트 자격 증명을 제공할 수 없는 경우 , 자격 증명 보안 모범 사례 가이드를 따라 Windows, Linux 및 SQL용으로 최소 권한의 사용자 지정 계정을 만들 수 있습니다 . Azure Migrate 프로젝트 개요 페이지에서 어플라이언스를 사용하여 검색을 클릭 하고 가상화 환경을 VMware로 선택합니다. 기기 이름을 지정하고 프로젝트 키를 생성하세요. 다운로드한 zip 파일을 사용하여 기존 서버를 어플라이언스 VM으로 변환 하거나 OVA 템플릿을 사용하여 새 어플라이언스 VM을 설정할 수 있습니다 . 이번 데모에서는 zip 파일을 다운로드하겠습니다. 관리자 권한으로 Azure Migrate 설치 프로그램 zip 파일을 실행하고 어플라이언스 유형을 VMware로 선택하십시오. 클라우드 유형과 어플라이언스를 공용 엔드포인트 또는 사설 엔드포인트 연결로 설정할지 여부를 선택하십시오. 압축 파일 실행이 완료되면 어플라이언스 구성 관리자 웹 앱이 자동으로 실행됩니다. 첫 번째 확인 사항은 필요한 Azure 엔드포인트에 연결할 수 있는지 확인하는 것입니다. 서버가 프록시 서버 뒤에서 실행 중인 경우 프록시 서버 정보를 추가하십시오. 인증형 및 비인증 형 프록시 서버가 모두 지원됩니다. 인증 기반 프록시를 사용하는 경우 해당 세부 정보를 입력할 수 있습니다. 시간 동기화 검사는 어플라이언스의 시간이 Azure와 동기화되어 검색이 원활하게 이루어지도록 보장합니다. 다음 단계는 이 어플라이언스를 Azure 마이그레이션 프로젝트에 등록하는 것입니다 . 여기에 Azure 포털 에서 생성한 프로젝트 키를 붙여넣어야 합니다 . 이 단계에서는 업그레이드 및 등록 작업이 동시에 실행됩니다. 기본적으로 자동 업그레이드가 활성화되어 있으므로 어플라이언스 관리에 대해 걱정할 필요가 없습니다. 다음 단계는 Azure에 로그인하는 것입니다 . 여기에 사용할 사용자 계정은 해당 기기를 프로젝트에 등록할 수 있는 충분한 권한을 가지고 있어야 합니다 . 필수 조건 확인이 완료되면 VMware VM을 관리하는 vCenter의 자격 증명을 추가하십시오 . 범위가 지정된 vCenter 계정을 사용하면 VMware 환경에서 특정 머신을 검색할 수 있습니다 . 사용자 이름과 비밀번호를 입력한 후, 알아보기 쉬운 이름을 자격 증명에 지정할 수 있습니다. 이제 vCenter 정보를 제공해야 합니다. vCenter의 FQDN 또는 IP 주소와 포트 정보를 제공하십시오. 기본적으로 어플라이언스는 443번 포트를 통해 vCenter와 통신합니다. vCenter가 다른 포트에서 수신 대기하는 경우 포트 번호를 지정할 수 있습니다 . 자격 증명에 사용된 친근한 이름을 선택하세요. 여기에 추가된 자격 증명은 항상 기기 내부에 저장됩니다. 자격 증명은 데이터 보호 API(DPAPI)를 사용하여 암호화되며 , 이는 안전한 256비트 암호화 방식입니다. 이제 vCenter를 검색 소스로 성공적으로 추가했습니다. 이러한 세부 정보를 통해 어플라이언스는 vCenter에서 머신을 검색할 수 있으며, 운영 체제 정보, 디스크 정보, CPU 정보 및 메모리 정보와 같은 서버 메타데이터가 Azure 포털에 표시됩니다. 소프트웨어 인벤토리의 심층 분석 , 에이전트 없는 종속성 분석, SQL 및 네트워크 부하 분석을 위해서는 서버 자격 증명을 제공해야 합니다. 여기에 다양한 유형의 자격 증명을 추가할 수 있습니다 . 서버 집합에 대한 도메인 자격 증명을 추가하거나 Windows, Linux 및 SQL 자격 증명과 같은 도메인 외 자격 증명을 추가할 수 있습니다. 이러한 자격 증명 풀을 제공하면 어플라이언스는 이러한 자격 증명을 지능적으로 매핑하여 소프트웨어 인벤토리를 검색하고, 종속성 분석을 수행하고, SQL 및 네트워크 부하를 검색합니다. 장비 관련 문제가 발생할 경우, 지원 요청을 접수할 필요 없이 진단 실행을 통해 자체적으로 문제를 진단할 수 있습니다 . 모든 자격 증명을 추가한 후 ' 검색 시작'을 클릭하여 검색 프로세스를 시작하세요. 데이터가 Azure 포털에 반영되는 데 약 10~15분이 소요됩니다. 에이전트 없는 마이그레이션을 수행하려면 동일한 VMware 어플라이언스를 사용할 수 있습니다. 다음 영상에서 발견된 인벤토리, 액션 센터에서 [음악] 문제 해결 방법, 그리고 사업 계획 수립 방법에 대해 알아보세요.
