## 스크립트

이번 GitHub Copilot CLI 초보자 강좌에서는 기본적인 사용법을 알아보겠습니다 . 이 글에서는 Copilot CLI를 설치하는 방법, 로그인하는 방법, 그리고 Copilot CLI에 보낼 첫 번째 요청 몇 가지를 살펴보겠습니다. GitHub Copilot CLI는 데스크톱에서 바로 사용할 수 있는 AI 에이전트입니다 . 다른 AI 코딩 도우미와 마찬가지로 C-Pilot CLI를 사용하여 코드베이스를 탐색하고, 질문하고, 코드를 생성할 수 있습니다. 하지만 에이전트를 특별하게 만드는 것은 코드 빌드 및 테스트 실행과 같은 작업을 자율적으로 수행할 수 있는 능력, 심지어 오류가 감지될 때 코드를 업데이트하여 자체 복구 기능까지 갖추고 있다는 점입니다. Copilot CLI는 개발자로서 기존 워크플로에 자연스럽게 통합되어, 대화 형으로 작업을 할당하고 할 일 목록의 다른 항목을 처리하거나, 생성된 결과와 애플리케이션에 미치는 영향을 확인하면서 반복적으로 빌드하고 변경 사항을 요청할 수 있습니다 . Copilot CLI를 사용하면 클라우드 기반 에이전트인 Copilot Coding Agent에 작업을 위임할 수도 있습니다 . Copilot CLI를 시작하려면 당연히 설치부터 해야 합니다. 이를 위한 핵심적인 크로스 플랫폼 방식은 npm install 명령어를 사용하는 npm을 통하는 것입니다 . github/copilot의 Tickg를 사용 하면 됩니다. Windget이나 Homebrew 같은 패키지 관리자를 사용하고 있다면 , 해당 도구를 통해 Copilot CLI를 설치할 수도 있습니다 . 자세한 내용은 해당 도구의 설명서를 참조하십시오 . 개인적으로는 Node를 통한 설치를 선호 하지만, 이는 제 안에 있는 웹 개발자 기질 때문일지도 모릅니다. 제품을 설치한 후에는 Copilot이라고 입력하여 실행할 수 있습니다. 사용할 수 있는 스위치는 여러 가지가 있으며 , 이에 대해서는 이후 영상에서 자세히 설명하겠습니다. 처음 사용하시는 경우 GitHub 계정으로 로그인해야 합니다 . 이것은 두 가지 일을 합니다. 첫째, 클라이언트를 Copilot 계정에 연결합니다 . 둘째, GitHub MCP 서버에 연결하여 GitHub의 리소스에 대한 액세스 권한을 부여합니다. 이 부분은 나중에 자세히 설명하겠습니다. Copilot을 사용하려면 Copilot이 폴더를 탐색하고 파일을 수정할 수 있도록 접근 권한을 부여해야 합니다. 이 기능을 이번 세션에만 적용하거나, 이후 세션에 적용하도록 선택할 수 있습니다. 이렇게 하면 특정 프로젝트에서 Copilot을 다시 실행할 때 매번 승인할 필요 없이 간편하게 실행할 수 있습니다. 그 과정이 완료되면, 자유롭게 대화를 시작하고, 질문을 하고, 코드나 다른 작업을 요청할 수 있습니다. 먼저 몇 가지 기본 사항부터 살펴보겠습니다. Copilot 측에 프로젝트 개요를 요청해 보겠습니다. Copilot은 탐색을 수행하고, 중요한 파일을 열고, 발견한 내용을 보고합니다. 새로운 엔드포인트를 생성해달라는 요청과 같은 코드를 요청할 수도 있습니다. 제가 요청을 하면 Copilot은 프로젝트를 다시 살펴보고 기존 문서와 예시를 찾아 최대한 기존 사례를 따르려고 노력할 것입니다. 다시 한번, Copilot은 파일 생성 권한을 요청할 것입니다. 제가 승인하겠습니다. 이제 새로운 엔드포인트가 생성되었습니다. 물론, 이것은 단지 시작일 뿐입니다. Copilot과 그 기능을 알아보는 가장 좋은 방법 중 하나는 Copilot에게 직접 물어보는 것입니다. 해당 프로그램의 문서를 살펴보고 프로그램과 상호 작용하고 , 실험하고, 탐색하는 방법에 대한 지침을 제공합니다 . 물론, 이 시리즈의 나머지 영상들도 시청하실 수 있습니다 . GitHub Copilot CLI를 최대한 활용하는 데 도움이 되도록 가장 일반적인 시나리오와 가장 강력한 옵션 몇 가지를 안내해 드리겠습니다 . 안녕하세요 여러분, GitHub Copilot CLI 초보자 시리즈에 다시 오신 것을 환영합니다. 오늘은 CLI의 다양한 모드, 즉 대화형 모드와 비대화형 모드에 대해 이야기해 보겠습니다. 자, 그럼 시작해 볼까요? Copilot CLI는 채팅처럼 코드를 주고 받기에 좋은 대화형 모드 와 간단한 일회성 프롬프트에 적합한 인라인 또는 비대화형 모드를 제공합니다 . 둘 다 살펴보겠습니다 . 대화형 모드로 들어가려면 copilot을 입력하고 Enter 키를 누르세요. Copilot은 일반적으로 이 폴더를 신뢰할지 묻는데, 이는 Copilot이 해당 파일의 읽기 및 수정이 가능하기 때문입니다. 대화형 모드에서는 " 이 프로젝트를 로컬에서 실행하려면 어떻게 해야 하나요?"라고 질문할 수 있습니다. Copilot에서 수동으로 실행하는 방법을 안내해 줄 텐데, 제가 직접 실행해 달라고 요청할 수도 있습니다. 그러면 Copilot이 프로젝트를 분석하고 서버를 시작합니다. 그러면 여기에서 프로젝트를 살펴보고, 변경하고 싶은 부분을 찾아, 같은 세션 내에서 C-Pilot을 사용하여 작업을 계속할 수 있습니다. 비대화형 모드로 전환하려면 Copilot을 종료하고 일반 명령줄로 돌아가야 합니다. 여기에서 copilot-p를 입력하고 copilot에게 " 이 저장소의 기능과 주요 폴더를 빠르게 요약해 주세요"와 같은 프롬프트를 제공할 수 있습니다. 그러면 Copilot이 프로젝트 파일을 살펴보고 명령줄 환경을 유지하면서 답변을 제공해 줍니다. 혼자만의 시간을 유지하면서 가끔씩 떠오르는 질문 이나 아이디어에 답하기에 아주 좋습니다 . 또 다른 팁은 슬래시 명령어를 사용하는 대신 여기에서 Copilot-Resume 명령어를 사용하여 이전 Copilot 세션을 재개할 수 있다는 것입니다 . 이미 작업할 세션을 알고 있다면 이 기능이 매우 유용합니다. 단 한 번의 명령으로 Copilot을 사용하여 개발을 시작할 수 있기 때문입니다 . 요약하자면, Copilot의 대화형 모드를 사용하면 코드를 주고받으며 장시간 작업을 진행할 수 있습니다. 빠른 안내를 위해 비대화형 모드 또는 인라인 모드를 사용할 수 있습니다. 안녕하세요 여러분, GitHub Copilot CLI 초보자 시리즈에 다시 오신 것을 환영합니다. 이 영상에서는 CLI의 슬래시(/) [음악] 명령어에 대한 전체적인 개요를 살펴보겠습니다 . 자, 그럼 시작하겠습니다. 슬래시 명령어는 Copilot CLI 내의 제어 인터페이스입니다. 이러한 기능은 사용자가 웹사이트를 조종하고, 변경 사항을 검사하고, 컨텍스트를 관리하고, 세션을 전환하고, 권한을 깔끔하게 유지하는 데 도움이 됩니다. 가장 인기 있는 것들 몇 가지를 빠르게 살펴보겠습니다. 모든 슬래시 명령어를 보려면 슬래시를 입력하고 스크롤하여 옵션을 살펴보세요. 사용 중인 모델을 변경하려면 슬래시(/)모델을 입력하세요. 여기에서 사용 가능한 모델 목록을 확인하고 작업에 따라 원하는 모델을 선택할 수 있습니다. 일부 모델은 리팩토링과 같은 빠르고 간단한 작업에 적합하고, 다른 모델은 기능 계획과 같은 심층적인 추론에 더 적합합니다 . 이 목록은 새로운 모델이 출시됨에 따라 변경될 수 있으며, 사용 중인 요금제와 조직에서 활성화한 기능에 따라 달라집니다. 오른쪽에 있는 숫자는 비용 배율을 나타내므로 어떤 모델이 플랜에 포함되고 어떤 모델에 추가 비용이 발생하는지 확인할 수 있습니다. 모델을 선택하셨으니 이제 토큰 사용량을 살펴보시는 것이 좋습니다. 그러려면 /context를 입력하세요. 이 화면에서는 남은 토큰 수와 시스템 여유 공간 및 버퍼 용량을 확인할 수 있습니다 . 더 넓은 공간을 확보하려면 /compact를 입력하여 대화를 요약하고 새 세션을 시작하지 않고 계속 진행할 수 있습니다. Copilot CLI의 가장 큰 장점은 사용량 한도에 도달하기 시작하면 자동으로 이 작업을 수행해준다는 것입니다. 프로젝트의 새로운 부분으로 넘어가는 경우 수동으로 수정할 수도 있고, /clear 명령어를 사용하여 완전히 처음부터 다시 시작할 수도 있습니다. 이전 세션을 재개하려면 /resume를 입력하세요. 이렇게 하면 로컬 세션과 원격 세션을 포함하여 이전에 참여했던 세션 목록이 표시됩니다 . 이전 세션에 들어가면 세션 기록이 표시되고, 동일한 컨텍스트에서 작업을 계속할 수 있습니다. Copilot을 사용하여 프로젝트를 변경한 후에는 ` /diff` 명령을 실행하여 변경 사항을 확인할 수 있습니다. Copilot을 종료하지 않고 다른 프로젝트 작업을 시작하려면 ` /CWD`를 입력하여 작업 디렉토리를 다른 저장소로 변경할 수 있습니다 . 또한, Copilot의 작업 범위를 프로젝트의 특정 부분으로 제한하려면 저장소 내에서 이 기능을 사용할 수도 있습니다 . 마지막으로, '이 세션에서 항상 파일 편집 허용'과 같은 특정 도구를 허용했지만 이를 취소하고 싶을 때가 있습니다. 예를 들어, 더 신중하게 관리하고 싶은 다른 저장소로 이동했다면 ` slashresetallowed tools` 명령어를 실행하여 도구 승인을 재설정할 수 있습니다. 이러한 슬래시 명령어를 사용하면 Copilot CLI를 더욱 세밀하게 제어할 수 있습니다. 그러니 목록에 있는 다른 항목들과 함께 꼭 확인해 보세요. 안녕하세요 여러분, GitHub Copilot CLI 초보자 시리즈에 다시 오신 것을 환영합니다. 오늘은 Copilot CLI만을 사용하여 새로운 기능을 기획하고, 구축하고, 코드를 검토해 보겠습니다. 자, 그럼 가봅시다. 이전 영상에서는 워크플로 전반에서 사용할 수 있는 다양한 슬래시 명령어를 살펴보았습니다 . 이제, 기능을 계획하고, Copilot을 사용하여 빌드하고, slashdelegate와 slashre를 사용하여 검토하는 데 순차적으로 사용할 수 있는 세 가지 강력한 도구를 살펴보겠습니다. 계획 모드 또는 계획 기능은 요구 사항을 미리 설정하여 Copilot이 원하는 것을 정확하게 구축하도록 하는 데 매우 유용합니다. Copilot은 코드를 작성하기 전에 명확한 질문을 하고 단계별 계획을 제시합니다. 저희 앱을 한번 살펴보겠습니다. 게임을 선택하고 ' 이 게임 지원하기'를 클릭해도 아무런 반응이 없는 것 같습니다. 개별 게임 보기 화면과 메인 페이지 모두에서 해당 기능을 지원했음을 보여줌으로써 이 기능이 제대로 작동하도록 하고 싶습니다. Copilot의 계획 모드를 사용하여 이것을 어떻게 구축할지 계획해 봅시다.
