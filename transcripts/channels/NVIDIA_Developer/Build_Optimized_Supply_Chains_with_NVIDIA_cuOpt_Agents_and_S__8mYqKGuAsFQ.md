## 스크립트

전통적으로 이러한 최적화 문제를 해결하는 것은 제약 조건을 수동으로 정의하고 솔루션을 반복적으로 검토하는 것을 의미했으며, 이로 인해 몇 주가 걸리고 결국에는 오래된 데이터가 남게 되었습니다 . NVIDIA에서는 에이전트 기반 워크플로와 CoOpt 의사 결정 인텔리전스 최적화 엔진을 기반으로 구축된 AI 우선 플래너를 사용하여 이 과정을 몇 분으로 단축했습니다. 이 튜토리얼에서는 NVIDIA CoOpt Supply Chain 에이전트를 설정하고 실행하는 방법을 보여드리겠습니다. 이 에이전트는 동일한 아키텍처를 기반으로 구축되었으며, 운영 연구 과학자, 공급망 계획 담당자 및 개발자가 배포하고 확장할 수 있도록 오픈 소스로 공개되었습니다. 작동 방식은 다음과 같습니다. 이는 오케스트레이터 하위 에이전트 패턴을 사용하는 다중 에이전트 워크플로입니다 . 오케스트레이터는 자연어로 된 문제 설명을 받아 특화된 LangChain 딥 에이전트에 위임합니다. 이 에이전트는 해당 설명을 선형 프로그래밍 또는 혼합 정수 프로그래밍 모델로 변환하고 GPU 가속 CoOpt를 사용하여 문제를 해결합니다. CoOpt 에이전트의 구조화된 스킬은 구문 분석, 모델링 및 계획 수립을 기본적으로 처리하여 완벽한 설명 가능성을 갖춘 최적화된 결정을 제공합니다 . 해당 에이전트는 완전한 오픈 소스이며 Brev Launchable로 제공됩니다 . 개발자 크레딧을 무료로 드립니다. 시작하기 전에 이것들을 챙기세요 . 설명란에 링크가 있습니다. 좋습니다, 그럼 시작하겠습니다. 첫 번째 단계는 환경을 설정하는 것입니다. Brev Launchable을 배포하세요 . 이를 통해 클라우드에 사전 구성된 GPU 환경이 제공됩니다 . CUDA, Docker 및 NVIDIA 컨테이너 툴킷이 모두 사전 설치되어 있으므로 사용자가 직접 구성할 필요가 없습니다 . 인스턴스가 준비되면 '노트북 열기'를 클릭하여 Jupyter 환경을 실행하세요. 두 번째 단계는 저장소를 복제하고 필수 구성 요소를 설정하는 것입니다. 먼저 buildon.nvidia.com에서 NVIDIA API 키를 발급받아야 합니다. 프로필에 로그인하고 API 키를 클릭한 다음 API 키를 생성하세요. 안전한 곳에 보관하세요. 탭을 닫으면 해당 내용을 복구할 수 없습니다. 노트북에서 CoOpt Examples 저장소를 복제하고 CoOpt 에이전트 하위 디렉터리로 이동합니다 . 이 저장소에는 에이전트 코드, 스킬, 구성 파일 및 샘플 시나리오 등 필요한 모든 것이 포함되어 있습니다. 다음으로 서브모듈을 초기화합니다. 이것들은 Nemo Agent Toolkit 채팅 UI에 필요합니다 . example.env 파일을 복사하여 환경 변수를 설정하세요 . API 키를 해당 파일에 저장하고 , 해당 키를 사용하여 NVIDIA 컨테이너 레지스트리에 인증하십시오. 3단계는 LLM을 NIM으로 배포하는 것입니다. 에이전트는 MiniMax M 2.5를 추론 모델로 사용하며 , NVIDIA NIM으로 로컬에 배포됩니다 . 새 터미널 탭을 열고 API 키를 NGC API 키로 내보내세요. 이는 동일한 키이지만 NIM 컨테이너에서 참조되는 이름만 다릅니다. 다음으로 Docker 명령어를 실행하여 NIM 컨테이너를 다운로드하고 시작합니다. 중요한 설정 변경 사항 하나는 컨테이너를 기본값인 8000번 포트 대신 8001번 포트에 매핑하는 것입니다. 다음 단계에서 CoOpt 에이전트가 8000번 포트를 사용할 예정이기 때문입니다. 컨테이너가 실행되면 간단한 curl 요청을 보내 LLM이 올바르게 응답하는지 확인하십시오. 그런 다음, config/config_deep_agent.yaml 에 있는 config.yaml 파일을 업데이트하여 LLM 기본 URL이 로컬 NIM 엔드포인트(포트 8001)를 가리키도록 하십시오. 이 YAML 기반 구성은 여기서 중요한 설계 결정 사항 중 하나입니다 . 에이전트, LLM 및 워크플로는 모두 구성 파일에 정의되어 있으므로 코드를 수정하지 않고도 동작을 조정할 수 있습니다 . 4단계, 에이전트 서비스를 구축하고 시작합니다 . 노트북으로 돌아가서 docker-compose 빌드 명령을 실행합니다. 이 패키지에는 전체 에이전트 워크플로, CoOpt 솔버, LangChain 심층 에이전트 오케스트레이션 및 Nemo Agent Toolkit 채팅 프런트엔드가 포함되어 있습니다. 그런 다음, 구성 명령을 사용하여 모든 것을 분리 모드로 불러오십시오 . 각 명령을 실행하는 데 몇 분 정도 소요되므로, 다음 명령으로 넘어가기 전에 모든 명령이 완전히 완료될 때까지 기다리십시오. 5단계, UI를 실행하고 최적화를 시작하세요. Brev 콘솔로 돌아가세요. 환경 탭으로 이동한 다음 액세스 하위 탭으로 이동합니다. 공유 가능한 URL 목록에서 Jupyter(8888번 포트), 채팅 UI(3000번 포트), Phoenix 트레이싱(6006번 포트) 등 세 가지 서비스가 실행 중인 것을 확인할 수 있습니다. 모든 포트가 정상 상태인지 확인한 후 채팅 UI를 열어보세요. 이것은 사용자가 에이전트와 상호 작용하는 Nemo Agent Toolkit 인터페이스입니다. 예를 들어 보겠습니다. CoOpt Examples 저장소에서 샘플 공급망 시나리오 중 하나를 복사하여 UI에 붙여넣으세요. 담당자는 계획 수립 및 실행에 착수할 것입니다. 문제를 해결하는 과정을 실시간으로 확인하실 수 있을 겁니다. 각 프롬프트 처리에는 몇 분 정도 소요되며, 최종 최적화된 솔루션은 자세한 설명과 함께 제공됩니다 . 에이전트의 추론 파이프라인, 중간 단계, 에이전트 결정, 모델 호출 등을 완벽하게 관찰하기 위해 개발자는 Phoenix 추적을 유연하게 설정할 수 있습니다. 미리 구성된 프롬프트를 사용하여 수요, 용량 또는 비용 입력값을 다양하게 변경해 보고 목표값, 의사 결정 및 제약 조건이 실시간으로 어떻게 반응하는지 관찰해 보세요. 작업이 완료되면 Brev 인스턴스를 삭제하여 모든 것을 깔끔하게 종료하십시오 . 여기에서 GitHub의 CoOpt 에이전트 스킬을 살펴보세요. 모듈식이며 확장 가능하므로 사용자 지정 제약 조건을 삽입하고 자체 데이터를 통합하거나 자체 기업 공급망 워크플로를 위해 더 광범위한 선형 프로그래밍 및 혼합 정수 프로그래밍 영역으로 확장할 수 있습니다 .
