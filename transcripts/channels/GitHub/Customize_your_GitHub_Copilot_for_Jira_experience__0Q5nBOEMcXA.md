## 스크립트

Jira에서 GitHub Copilot 클라우드 에이전트를 사용할 때 두 단계의 사용자 지정 옵션이 있습니다 . 첫째, 해당 특정 실행을 특정 방식으로 처리해야 하는 경우에 대한 티켓별 지침입니다. 둘째, 프로젝트 전체에 적용되어야 하는 모든 사항에 대한 공간 수준별 지침입니다 . 티켓별 지침부터 시작하겠습니다. 리포지토리에 Copilot 에이전트가 두 개 이상 구성되어 있는 경우 특정 티켓을 처리할 에이전트를 선택할 수 있습니다. Jira 댓글에 GitHub Copilot을 언급하고 사용할 에이전트 이름을 지정하면 Copilot이 해당 에이전트를 사용하여 실행합니다. 따라서 이것이 문서 관련 작업이라면 문서 에이전트로 전달하십시오. API 리팩토링 관련 작업이라면, 해당 작업을 위해 설정된 담당자에게 전달하세요. 이제 모델별 맞춤형 지침도 제공할 수 있습니다 . 또한 사용자 지정 필드 지원을 통해 사용자 지정 에이전트와 같은 사용 가능한 옵션을 안내받을 수 있으므로 정확한 이름을 기억할 필요가 없습니다. 이 설정은 GitHub에 접속하여 설정을 변경할 필요 없이 Jira 티켓에서 바로 진행할 수 있습니다 . Copilot이 시작되면 채팅 패널에 작업 내용이 실시간으로 표시되고 상단에 전체 상담원 세션을 열 수 있는 링크가 나타납니다. 작업이 완료되면 Copilot은 Jira 티켓에 댓글을 달아 PR이 검토 준비가 완료되었음을 알립니다. 후속 댓글은 댓글을 어디에 남기느냐에 따라 달라집니다. 채팅 패널의 댓글은 현재 PR을 업데이트하는 반면, Jira 티켓 자체에 댓글을 달면 새 실행이 시작되고 새 PR이 열립니다. 공간 레벨 지침은 티켓마다 변경되지 않는 모든 항목에 적용됩니다 . 가장 흔한 예는 저장소입니다. 이 Jira 공간의 모든 티켓이 동일한 GitHub 저장소로 작업을 전송해야 하는 경우 , 공간 수준에서 한 번만 고정하세요 . 이제 해당 공간의 모든 트리거는 기본값이 그 값입니다. 티켓별 설정이 필요 없고, 어떤 저장소를 사용해야 하는지 기억할 필요도 없습니다 . 언어, 분기 명명 규칙 또는 기본 에이전트와 같은 다른 영구적인 지침에도 동일한 위치를 사용할 수 있습니다. 한 번만 설정하면 Copilot이 프로젝트 전체에 적용합니다. 특정 실행을 제어해야 할 때는 티켓을 사용하고, 전체 프로젝트에 적용될 기본값에는 공간 수준의 지침을 사용하십시오 . Copilot은 두 가지 모두를 읽으므로 팀은 매번 동일한 지침을 반복하지 않고도 워크플로를 맞춤 설정할 수 있습니다 .
