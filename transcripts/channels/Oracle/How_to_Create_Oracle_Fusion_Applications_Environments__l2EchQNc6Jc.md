## 스크립트

이 튜토리얼에서는 환경 설정 마법사를 사용하여 Oracle Fusion Applications 환경을 생성합니다 . Fusion Applications 환경을 생성하면 구현을 시작하는 데 필요한 운영 및 테스트 환경이 준비됩니다 . 이 시나리오에서는 이미 Oracle Fusion Applications 구독을 활성화하고 Oracle Cloud 계정에 로그인했습니다 . 이제 Fusion Applications 환경을 프로비저닝할 준비가 되었습니다. OCI 콘솔의 Fusion Applications 페이지에서 빠른 시작을 선택하여 환경 설정 마법사를 엽니다. 내 환경 패밀리 정보를 검토하고 기본 이름을 유지한 다음 구독 정보를 검토하고 계속 진행하려면 해당 구독을 선택하세요. 이제 구독 세부 정보를 검토하고 환경을 생성할 Oracle Cloud 지역을 선택하겠습니다. Oracle Fusion Data Intelligence도 사용할 계획이라면 Oracle은 두 서비스에 동일한 지역을 선택할 것을 권장합니다 . 고급 옵션에서 환경 패밀리가 생성될 구획을 선택하겠습니다. 이 예시에서는 루트 구획을 사용하겠습니다. 유지보수 일정을 검토하고 기본 설정을 유지한 후 다음 단계로 진행하겠습니다. 이제 환경을 설정하겠습니다. 기본 환경 이름을 그대로 유지하겠습니다. 제 구독에는 프로덕션 환경 과 테스트 환경이 모두 포함되어 있으며, 두 환경은 함께 프로비저닝될 예정입니다. 다음으로 Fusion Applications 관리자를 구성하겠습니다. 모든 환경에 동일한 관리자를 할당하거나, 프로덕션 환경과 테스트 환경에 서로 다른 관리자를 지정할 수 있습니다. 프로비저닝이 완료되면 관리자는 계정을 활성화하고 비밀번호를 생성하라는 이메일을 받게 됩니다. 나머지 고급 옵션들을 검토해 보겠습니다 . 기본 DNS 접두사를 그대로 사용하겠습니다. 그러면 제 환경에 맞는 서비스 URL이 자동으로 생성됩니다. 기본적으로 Oracle 관리형 암호화가 사용됩니다. 설정을 검토한 후, 환경을 생성하기 전에 모든 설정을 다시 확인하기 위해 '다음'을 선택하겠습니다 . 모든 것이 올바른 것 같으니, 프로비저닝을 시작하기 위해 제출을 선택하겠습니다. 오라클은 먼저 환경 패밀리를 생성한 다음 개별 프로덕션 및 테스트 환경을 프로비저닝합니다. 프로비저닝이 완료되면 관리자는 각 환경에 대한 별도의 활성화 이메일을 받게 됩니다. 계정을 활성화한 후에는 구현을 시작할 수 있습니다. 배포가 제대로 되었는지 확인하기 위해 OCI 콘솔에서 애플리케이션을 선택한 다음 Fusion Applications를 선택하여 Fusion Applications 페이지로 돌아가겠습니다. 개요 페이지에는 내 환경 패밀리가 표시됩니다. 환경 페이지에는 각 Fusion Applications 환경의 상태, 구획 및 애플리케이션 URL이 나열됩니다. Fusion Applications 환경을 구축했으므로 이제 구현을 시작하고 필요에 따라 동일한 클라우드 계정에서 추가 Oracle 서비스를 프로비저닝할 수 있습니다 . 아래 링크에서 Oracle Fusion Applications 배포 관련 추가 자료 및 튜토리얼을 찾아보세요.
