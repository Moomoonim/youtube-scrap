안녕하세요 여러분, 반갑습니다. 이 영상에서는 웹 애플리케이션의 마이그레이션 및 현대화 계획을 수립하는 방법을 살펴보겠습니다 . 이전 영상들을 보셨다면 , 일반적인 이민 과정이 결정, 계획, 실행의 단계로 나뉜다는 것을 이미 알고 계실 겁니다 . 이 영상에서는 웹 애플리케이션 평가에 대해 다룹니다. 웹 애플리케이션 평가는 일반적으로 마이그레이션 과정의 계획 단계에서 진행됩니다. 웹 앱 평가는 대상 서비스 전반에 걸친 평가를 철저히 분석하고 마이그레이션 준비 상태 및 관련 비용을 고려하여 Azure 앱 서비스와 Azure Kubernetes 서비스 모두에 대한 준비 상태 점검을 수행합니다. 본 평가는 웹 애플리케이션의 최적 마이그레이션 경로에 대한 정보에 기반한 권장 사항을 제공합니다 . 평가 보고서에는 해결해야 할 문제점이나 경고 사항에 대한 자세한 정보와 해결을 위한 권장 사항도 포함되어 있습니다. 웹 애플리케이션 마이그레이션을 평가하고 효과적으로 계획하기 전에, 먼저 사전 조사( discovery)를 성공적으로 완료하는 것이 필수적입니다. 성공적인 검색을 위해서는 인프라 내 모든 서버에서 소프트웨어 인벤토리가 수행되었는지 확인하십시오. 또한, 검색 기능에 영향을 미치는 문제가 있는지 작업 센터를 검토하고 권장 조치를 따라 검색 문제를 해결하십시오. 웹 애플리케이션 평가를 생성하는 데에는 여러 가지 시작점이 있습니다. 개요 화면 이나 전체 인벤토리 화면에서 평가 생성 마법사로 이동할 수 있습니다. 웹 앱을 필터링하고 선택한 다음 '평가 생성'을 클릭하거나, 인벤토리에서 새로운 웹 앱별 보기로 이동할 수 있습니다. 보시다시피, ASP.NET 웹 애플리케이션과 Java Tomcat 웹 애플리케이션 모두를 발견했습니다. 이 페이지에서 웹 애플리케이션 평가부터 시작해 보겠습니다. 평가할 웹 애플리케이션을 선택하고 ' 평가 생성' 버튼을 클릭하세요. 평가에 사용할 친근한 이름을 지정해 주세요. 미리 선택된 웹 앱들이 이미 추가된 것을 확인할 수 있습니다. 다음으로, 요구 사항에 맞게 평가 설정을 사용자 지정할 수 있습니다. 대상 위치에 따라 평가에 사용할 Azure 지역이 결정됩니다 . 기본 환경을 사용하면 프로덕션 및 개발 테스트 워크로드를 별도로 평가할 수 있습니다. 기본적으로 평가는 사용량에 따라 지불하는 요율을 사용하여 생성됩니다. 마이크로소프트와 기업 계약을 체결하셨다면 기업 계약을 선택하십시오 . 성능 기반 또는 온프레미스 적정 규모 산정 기준 중에서 선택할 수 있습니다. 성능 기반 크기 조정은 워크로드의 리소스 활용률과 구성 속성을 고려하여 Azure에 적합한 크기로 조정하는 반면, 온 프레미스 크기 조정은 온 프레미스 워크로드의 구성 속성만 고려합니다. 고급 설정에서는 웹 애플리케이션을 평가할 대상을 선택할 수 있습니다 . ' 기본값 편집'을 클릭하여 설정을 확인해 보겠습니다. 대상별 설정을 구성할 수 있습니다 . 모든 것이 괜찮아 보이면 ' 만들기'를 클릭하세요. 평가 결과는 몇 분 안에 검토 가능합니다. 미리 만들어진 평가로 이동해 보겠습니다. 보시다시피 서버는 20개, 웹 애플리케이션은 35개, 그리고 평가 대상 웹 서버는 20개입니다. 앞서 언급했듯이 웹 애플리케이션 평가는 지원되는 모든 Azure 대상 서비스를 사용하여 웹 애플리케이션을 평가합니다 . 보시다시피, 35개의 웹 앱 중 15개는 준비가 완료되었고 20개는 조건부로 준비가 완료되었으며 예상 월 비용은 약 947달러입니다. '세부 정보 보기' 옵션을 클릭하여 권장 마이그레이션 경로에 대한 자세한 내용을 살펴보겠습니다. 권장되는 마이그레이션 경로는 34개의 웹 앱을 앱 서비스 코드로, 1개의 웹 앱을 AKS로 마이그레이션하는 것을 제안합니다. 웹 앱을 앱 서비스 코드로 변환하는 권장 사항을 살펴보겠습니다. 앱 서비스 코드 보고서를 보면 34개의 웹 앱 중 15개만 앱 서비스 코드로 마이그레이션할 준비가 된 것을 알 수 있습니다. 권장되는 스큐 크기는 프리미엄 v3이며 예상 비용은 월 620달러입니다 . 목표 별 평가를 자세히 살펴보려면 해당 목표 탭으로 이동하여 관련 세부 정보를 확인할 수 있습니다. 예를 들어, 앱 서비스 컨테이너 탭을 확인해 보겠습니다. 35개의 앱 중 15개는 준비가 완료되었고, 19개는 조건부로 준비가 완료되었으며, 1개는 아직 준비가 되지 않았습니다. 아직 준비되지 않은 웹 애플리케이션의 경우 , 권장되는 도구는 서버 마이그레이션입니다. 즉, 준비 상태 탭을 클릭하여 해당 서버를 들어 올려 이동해야 합니다 . 앱이 준비되지 않은 상태로 표시되는 마이그레이션 문제를 확인할 수 있습니다. 다음은 조건부로 준비 완료로 표시된 또 다른 앱입니다 . 앱을 클릭하면 모든 대상에 대한 앱별 준비 상태에 대한 자세한 정보를 확인할 수 있습니다. 개요 페이지로 다시 돌아오면 다른 대상 서비스별 평가 보고서를 검토할 수 있습니다. 또한 개요 페이지에서 평가 보고서를 내보내 오프라인에서 확인할 수 있습니다. 이 비디오에서는 Azure Migrate의 새로운 웹 애플리케이션 평가 기능을 살펴보았습니다. Azure 평가에서 웹 앱 생성 및 다양한 옵션에 대해 논의하고, 평가 보고서를 검토하여 권장 마이그레이션 경로와 Azure에서 웹 앱을 실행하는 데 필요한 예상 비용을 파악했습니다. 마이그레이션 여정의 다음 단계는 실행입니다. 웹 서버를 Azure VM으로 마이그레이션하기 위한 검색 및 평가가 완료되면 Azure Migrates의 기본 리프트 앤 시프트 기능을 활용하여 ASP.NET 또는 Java Tomcat 웹 앱을 앱 서비스 코드로 현대화할 수 있습니다. 앱 서비스 마이그레이션 지원 도구를 사용하면 웹 앱을 앱 서비스 컨테이너 또는 Azure Kubernetes 서비스로 마이그레이션할 수 있습니다 . 이 영상을 시청해 주셔서 감사합니다.
