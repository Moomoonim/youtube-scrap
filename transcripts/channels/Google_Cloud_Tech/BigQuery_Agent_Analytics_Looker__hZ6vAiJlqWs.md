## 스크립트

팀에서 에이전트 개발을 시작하셨군요 . 이제 그들은 여러분에게 와서 성과, 사용량 및 비용을 측정하고 추적하는 방법에 대해 묻고 있습니다. Looker는 최근 원격 측정 데이터 위에 설치되어 대시 보드와 탐색 기능을 바로 사용할 수 있도록 해주는 BigQuery 에이전트 분석 블록을 개발했습니다. 이제 Agent Development Kit 또는 LangChain을 사용하여 구축하면 원격 측정 데이터를 BigQuery로 바로 스트리밍할 수 있습니다 . 그런 다음 LookML을 사용하여 모델링할 수 있습니다. 처음 설정을 완료하려면 약간의 사전 준비 작업이 필요합니다. 에이전트를 BigQuery API를 통해 BigQuery에 연결해야 하며 , 이를 위해서는 API를 활성화해야 합니다. 그리고 로컬 환경에서 개발하는 경우 앱을 인증해야 합니다. 또는 서비스 계정에 올바른 권한을 부여해야 합니다. 저것은 BigQuery Job User이고 BigQuery Data Editor입니다. 다음으로 BigQuery 에서 클라우드 프로젝트와 데이터 세트 ID를 설정해야 합니다 . 그런 다음, ADK 앱의 플러그인 구성에서 해당 내용을 확인하세요 . 좋아요. 그런 다음 앱을 실행하고 몇 가지 프롬프트와 응답을 받아 BigQuery 데이터 세트로 데이터가 스트리밍되는 것을 확인합니다 . 한번 확인해 봅시다. 보시다시피, 스키마는 플러그인에 의해 채워지며, 플러그인을 추가로 구성하여 데이터 세트로 스트리밍할 데이터를 정확하게 지정할 수 있습니다. 데이터를 확인하려면 쿼리를 실행하세요. 이 쿼리를 실행하면 사용자 메시지가 수신되고, 에이전트가 시작되고, 요청이 LLM으로 전송되었으며, 응답까지 확인할 수 있습니다. 이제 원격 측정 데이터가 BigQuery로 스트리밍되므로 Looker 블록을 연결해야 합니다. 먼저 BigQuery 와 Looker를 연결합니다. 그런 다음 마켓플레이스로 이동하여 연결, 데이터가 호스팅되는 클라우드 프로젝트, 데이터 세트 ID 및 테이블( 일반적으로 agent_events로 기본 설정됨)을 지정하여 마켓플레이스를 통해 블록을 직접 설치합니다. 이 시점에서는 LookML에 대한 지식이 없어도 이 Looker 블록을 바로 사용할 수 있습니다. 설치 및 연결 후 여기서 멈추기만 하면 비즈니스 사용자에게 바로 대시보드 액세스 권한을 부여할 수 있습니다 . 하지만 저처럼 이 코드 블록을 다른 프로젝트로 직접 가져오고 싶다면 , 프로젝트 폴더에 있는 매니페스트 파일의 원격 종속성 매개변수에 GitHub 저장소의 URL과 브랜치 이름을 지정하여 가져오면 됩니다 . 해당 블록의 파일은 항상 읽기 전용으로 가져옵니다. 따라서 뷰와 탐색기를 모델 파일로 그대로 옮기려면 뷰와 탐색기를 더욱 정교하게 다듬고 확장해야 합니다 . 또한 대시보드 파일은 원래 모델에 연결되어 있으므로 해당 파일을 모두 로컬 프로젝트로 복사해야 합니다. 그런 다음, 그들이 가리키는 모델을 바꾸고 고유한 이름을 지정하세요. 블록을 확장하는 방법을 예시로 보여주기 위해 AG ADK 프로젝트를 통해 전달되는 사용자 프롬프트를 살펴보고 그 작동 방식을 보여드리고 싶었습니다. 그래서 Looker에 BigQuery의 JSON 값 구문을 사용하여 JSON에서 해당 값을 파싱하는 차원을 작성했습니다 . 그런 다음 LookML의 구문을 사용하여 뷰를 확장했습니다. 변경 사항을 적용하고 배포했더니, 쿼리에서 프롬프트 값을 확인할 수 있게 되었습니다. 다른 데이터 필드에서도 비슷한 작업을 수행할 수 있습니다 . 또한, 대시보드 에이전트도 있습니다. 대시 보드와 대화를 나눌 수 있으며, 적용한 모든 필터가 고려됩니다. 이를 통해 비즈니스 사용자는 탐색과 동시에 통찰력을 얻을 수 있으며, 더 복잡한 비즈니스 인텔리전스 질문에 시간을 절약할 수 있습니다. 대시 보드 에이전트가 어떻게 작동하는지 아주 간단한 예시를 보여드리겠습니다. 제 작은 앱에서 " 제 앱을 5일 동안 사용했을 때 토큰 사용량 예측치는 어떻게 되나요?"라고 물어볼 거예요. 이 함수는 아주 간단한 계산을 하지만, 자세히 살펴보면 어떤 다른 정보를 제공할 수 있는지 알 수 있습니다 . 마켓플레이스에서 에이전트 분석 도구를 설치하여 빠른 대시보드를 사용해 보거나, 직접 확장하여 Looker 커뮤니티에 어떻게 구현했는지 공유해 주세요. 그러면 모든 사용자가 해당 정보를 활용할 수 있습니다. 괜찮은. 들어주셔서 감사합니다. 이 블록을 유용하게 사용하시길 바랍니다. 곧 얘기해요.
