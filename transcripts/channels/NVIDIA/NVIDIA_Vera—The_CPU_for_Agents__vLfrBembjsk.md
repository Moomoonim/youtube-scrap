## 스크립트

에이전트형 AI는 CPU의 역할을 변화시킵니다. 이제 CPU는 지휘자이고 GPU는 오케스트라입니다. 기존 CPU(음악용)는 소켓당 코어 수를 최대화하는 방식으로 다른 시대에 맞춰 설계되었습니다. 그것들을 잘게 쪼개고 , 가상화하고, 시간 단위로 음악을 대여하세요. 에이전트 시대에 접어들면서 CPU는 GPU 활용률의 병목 현상이 되어 토큰 처리량, 지연 시간 및 사용자 경험에 직접적인 영향을 미칩니다. [음악] 엔비디아 베라는 에이전트 루프를 위해 설계된 CPU로 , 엔비디아의 맞춤형 [음악] 데이터 센터 CPU 코어와 확장 가능한 일관성 패브릭을 결합하여 성능 코어와 대역폭의 균형을 최적화함으로써 AI 팩토리의 생산성을 극대화합니다. Vera의 핵심에는 최신 데이터 센터 워크로드, 분기 중심의 Python 런타임, 도구 호출 및 샌드박스 코드 실행을 위해 구축된 Nvidia Olympus [음악] 코어가 있습니다. 각 코어는 처리량에 맞게 조정됩니다. 주기당 두 개의 분기를 평가하는 신경 분기 예측기 . 10개 폭의 디코딩 엔진은 매 주기마다 더 많은 작업을 처리합니다 . 대형 엔진이 제대로 작동하지 않아 명령이 계속 전달됩니다. 새로운 그래프 엔진을 통해 다음 데이터 경로를 예측하는 고급 프리페처. 하지만 빠른 코어는 데이터(음악)가 정확하고 제시간에 도착할 때만 의미가 있습니다. Vera는 대역폭을 저하시키지 않고 여러 오류를 동시에 수정하면서 LPDDR5X 메모리를 사용하는 최초의 CPU입니다 . Vera는 X86 대비 최대 메모리 지연 시간을 40% 낮춰 데이터 검색, 분석 및 샌드박스 실행 과정에서 코어에 필요한 데이터를 제때 공급합니다. 엔비디아의 2세대 확장 가능한 일관성 [음악] 패브릭은 메모리와 I/O를 위한 별도의 다이를 사용하여 88개의 올림푸스 코어를 하나의 단일체 메시로 통합합니다. 코어가 칩렛에 분산되지 않아 기존 [음악] CPU보다 코어 간 통신 속도가 50% 더 빠릅니다 . 또한 메모리 일관성을 유지하는 NVLink 칩 간 연결은 GPU를 패브릭에 직접 연결합니다. GPU 외에도 NVLink 칩 간 연결을 통해 Vera는 여러 소켓으로 확장하여 CPU 간에 엄청난 대역폭을 제공할 수 있습니다. Vera는 x86 CPU 대비 에이전트 샌드박스 성능을 1.8배 향상시킵니다 . 독립형 Vera 랙은 에이전트 샌드박스, 도구, 코드 및 데이터 파이프라인을 실행합니다. Vera는 Rubin GPU와 긴밀하게 연동되어 워크플로우를 가속화합니다. NVIDIA Vera BlueField [음악] 4 STX는 컨텍스트 메모리와 AI 스토리지를 지원합니다. 컴퓨팅, 네트워킹, 스토리지까지, Vera는 에이전트 시대의 CPU입니다.
