Do you want to be able to create Jira issues when new tickets are created in Zendesk? Let me show you how to do it automatically using Zapier. To do this task, I want to create a Zap. In Zapier, a Zap is a simple way to connect different apps and make them work together automatically. With Zapier's no-code app builder, I can create my Zap from scratch or with Zapier's new AI Co-pilot, I can just describe what I want. I can say something like, "When a new Zendesk ticket is created, create a Jira issue." and it'll build the workflow for me. My Zap will then be created. On the left-hand side, the Co-pilot will tell me the steps that I need to take to complete my Zap. And if I need to tweak my prompt or troubleshoot anything, I can just type right into the Co-pilot. It walks me through the process. To start, select your trigger. Our trigger is an event that starts any Zap. In this case, our trigger is Zendesk. I'll need to log into my account before configuring my step. To do this, click on the account tab. The first time we set up Zendesk, we'll need to connect it with Zapier. If it's your first time logging in, a link will pop up for you to log into your account and connect your API token key. To connect your key, go to the apps and integrations tag under your settings page and click API token. From there, you can select add API token. Give your token a description, hit save, and then copy and paste the token into the authorization screen. And don't worry, Zapier will only perform the actions you tell it to in a given Zap. We'll then need to fill out our trigger app's data. Select the form you'd like to use for your Zap and then click continue. In order to finish setting up the trigger, we'll need to pull in some example data. To do that, click test trigger. Testing our steps ensures that the data we entered is working. You're all set with the first step. Click continue with selected record and we can start creating our action. In Zapier, an action is the task that happens automatically after our trigger is set off. Your action event should already be set. So, log into your Jira account just like you did with your Zendesk account. From there, we can start to assign our trigger app's data into the action app's field. Co-pilot will automatically fill out our configuration step with our Zendesk data. And if my data isn't there, I can ask the Co-pilot to fill it out for me. With that, I can select from the drop-down menu or add my own text to customize my Zap even further. Once that's complete, I'll test the step. Great. Let's check our Jira account to make sure this works. Yep, there it is. This is ready to turn on. Now, whenever a ticket is created in Zendesk, an issue will be created in Jira as well.
