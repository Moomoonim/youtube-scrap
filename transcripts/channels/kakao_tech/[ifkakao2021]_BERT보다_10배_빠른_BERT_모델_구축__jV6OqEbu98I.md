멀트보다도 수천배 더 큰 GPT3라는 모델이 출시되었는데 텍스트 분류는 물론 글을 짓거나 간단한 웹 프로그래밍도 할 수 있다고 합니다. 카카오 추천팀에서 이러한 텍스트 분류 문제에 대한 접근 방식은 당시에 성능이 좋다고 알려졌던 CNN, LSTM 또는 패스텍스트 등을 응용한 모델을 구현하는 것이었습니다. 실제로 여러 서비스에서 사용 중에 있고 그 중에서 가장 성능이 좋았던 모델을 추가로 고도한 모델이 기존 쇼핑아웃 카테고리 분류기입니다. 저는 이러한 텍스트 분류 문제를 어떻게 접근할지 고민을 해 봤는데요. 카카오 추천 팀에서는 매년 카카오 아레나라는 인공지능 대회를 주최하는데 아레나 1의 대회가 쇼핑 카테고리 분류 문제로 해당 대회 수상작들은 이미 쇼핑아웃 데이터 일부에 대해 검증이 되었습니다. 따라서 수상 모델들을 기반으로 빠르게 베이스라인 모델 구현하고 그 이후에나 트랜스포머 기반의 소타 모델들을 시도하는 방법으로 진행했습니다. 아레나 수상 기법들을 참고하여 베이스라인 모델 구현을 시작했고 학습 파이프라인은 다음과 같습니다. 아레나 데이터로 주어진 상품명 약 800명에 대해 전 처리 단계에서 특수 문자를 제거해 주고 토크나이징을 진행합니다. 그리고 레즈넷 50으로 학습된 이미지 피처와 함께 인풋으로 만들었습니다. 학습은 GPU V백장으로 진행했으며 정확도와 F0.5 스코어를 산출하여 정량 평가를 진행하고 정답과 예측값을 비교하는 정성 평가 또한 진행했습니다. 명령어 하나로 전철에부터 평가까지의 프로세스가 진행되었고 모든 실험에 대해서는 하이퍼파라미터 세팅과 로그들을 남겼는데 그 이유는 라이브러리 버전만 되어도 나중에 제한이 안 될 가능성이 있었기 때문입니다. 실험 결과 단일 모델 기준 2위와 3위 모델 사이 앙상불 후에는 1위 수상작을 상해하는 스코어가 나왔습니다. 아레나 1 대회 수상 모델들을 살펴보면 센텐스 피스 토크이저의 공통적으로 서월스 기반에 인베딩해 LSTM이나 LSTM의 어텐션을 추가한 구조를 사용했고 드라웃이나 nor라이션을 모델 사이즈를 제한했고 외부 데이터를 사용하지 않았는데 이렇게 동일하게 한 이유는 대회 수상자 기법들을 잘 적용했는지 확인하기 위해서 했습니다.이 시점부터는 아레나 대회 데이터는 더 이상 사용하지 않고 실제 쇼핑아웃 데이터 수억로 넘어갔습니다. 그다음 단계로는 최신 소타 모델들을 실험했습니다. 최신 모델들로는 볼트를 조금 더 업그레이드한 모델들이 있는데요. 바로 학습시키기에는 문제가 있었습니다. 첫 번째로 볼트의 경우 ge 모델과 그것보다 작은 베이스 모델이 있는데이 베이스 모델만 해도 학습 시간이 이렇게 기준 한 달이 걸렸기 때문에 실험 단위를 너무 크게 잡아야 했습니다. 두 번째로 월트는 일반적인 언어 분야에서는 성능이 입증되었지만 쇼핑이나 금융, 바이오 같은 특정 분야에서는 성능이 떨어지고 그 이유는 멀트가 위키디아 같은 일반적인 텍스트에 대해 사전 훈련이 되었기 때문입니다. 그에 대한 추가 근거로 지금까지 금융이나 과학 같은 특정 도메인에 속하는 데이터들에 대해 텍스트 분류 모델을 만들었을 때 오히려보다 트랜스포머 아니면 LSTM 모델들의 성능이 비슷하거나 더 좋았던 사례들이 많았었습니다. 따라서 사전 훈련된 웨이트를 그대로 사용하지 않고 인코더 빠지기도 했고 상품 데이터는 넥센텐스 프레딕션 사전 훈련을 하기에는 부적합하기 때문입니다. 이렇게 만들어진 모델의 경우 초기 구연시 학습 속도가 이전에 만들었던 아레나 대회 모델 대비 7.5배가 5배가 더 빨라졌습니다. 학습 속도가 빨라진 이유는 어낸 계열 모델은 첫 번째 토큰부터 끝까지 계산해서 인코딩을 하지만 트랜스포머는 행렬 곱셈으로 한 방에 계산하여
