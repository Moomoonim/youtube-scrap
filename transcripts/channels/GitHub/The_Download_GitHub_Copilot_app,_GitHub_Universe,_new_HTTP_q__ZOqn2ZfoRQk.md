## 스크립트

GitHub Universe가 이번 10월 샌프란시스코에서 개최됩니다 . GitHub 이슈와 Copilot 앱에 대한 몇 가지 새로운 업데이트가 있습니다 . 자바스크립트에 새로운 기능들이 추가되었고, HTTP에는 10년 만에 처음으로 새로운 메서드가 생겼습니다. 이 모든 것과 그 이상의 이야기를 이번 The Download 에피소드에서 만나보세요. 우리가 돌아왔습니다. 최신 개발자 소식과 오픈 소스 프로젝트를 다루는 프로그램, The Download의 새로운 에피소드에 오신 것을 환영합니다. 좋아요 와 구독 부탁드립니다. 제 이름은 캐시디이고, 깃허브의 개발자 경험 팀에서 일하고 있습니다. 자, 그럼 시작해 볼까요? GitHub Copilot 앱은 이제 모든 Copilot 플랜에서 사용할 수 있으며, 새로 추가된 플랜에는 Copilot 무료 플랜과 GitHub 교육 플랜이 포함됩니다. macOS, Windows 및 Linux에서 사용할 수 있으며 GitHub 계정으로 로그인하여 데스크톱에서 몇 번의 클릭만으로 세션을 시작할 수 있습니다 . 이 앱은 자체 키를 사용하는 것도 지원하므로 GitHub Copilot 하네스와 그에 따른 모든 유용한 기능을 활용하여 자체 모델 제공업체에 대해 세션을 실행할 수 있습니다. 다른 릴리스 소식으로는 GitHub 이슈 필드를 이제 일반 사용자에게 제공합니다. 이슈 필드는 구조화된 형식의 메타데이터를 이슈에 추가하여 팀에서 우선 순위, 소요 시간, 날짜 또는 사용자 지정 값 등을 저장소 전반에 걸쳐 추적할 수 있도록 합니다. 5월 공개 이후 4만 개 이상의 기관에서 이슈 필드를 채택했으며, 이제 모든 분들이 이용하실 수 있게 되어 매우 기쁩니다. 이제 이슈 필드에 저장소 이슈 목록이 직접 표시되므로 각 이슈를 개별적으로 열지 않고도 메타데이터를 확인할 수 있습니다 . 이러한 필드는 GitHub의 MCP 서버를 통해서도 사용할 수 있으므로, 도구를 사용하여 이슈를 생성하거나 업데이트할 때 필드 값을 읽고 설정할 수 있습니다. 마지막으로, GitHub Universe가 10월 26일과 29일에 샌프란시스코에서 개최됩니다. 이 행사에서는 제품 출시, 전문가 주도 워크숍, 라이브 데모, 그리고 개발자 및 오픈 소스 관리자와 소통할 수 있는 기회가 제공될 예정입니다. 현장 구매 시 조기 예매 할인이 적용되며, 온라인 예매는 무료입니다. 그래서 만약 여러분이 탭 사용과 스페이스 사용에 대해 확고한 의견이 있다면, 어느 쪽 입장이든 상관없이 이야기를 나눌 사람들을 많이 찾을 수 있을 겁니다 . 그곳에서 뵙기를 바랍니다 . 오픈 소스 소식으로는 지난달 최신 ECMAScript 언어 사양이 공식 승인되었습니다. 이것을 자바스크립트라고도 알고 계실 수 있습니다. 배열에 대한 비동기 메서드, 오류에 대한 `iserror`, `math.sumprecise` 등과 같은 새로운 메서드가 추가되었습니다. 언어가 하룻밤 사이에 획기적으로 변모하는 화려한 변화는 아니지만, 정말 흥미롭고 여러분의 골칫거리를 덜어줄 수 있는 진전입니다 . 개인적으로 가장 마음에 드는 기능은 get 또는 insert라고 불리는 새로운 메서드인데 , 이 메서드를 사용하면 " 이 항목이 존재하면 가져오고, 그렇지 않으면 설정"과 같은 조건부 작업을 하나의 함수로 처리할 수 있습니다. 정말 멋지네요. 그리고 마지막으로, [음악] 매주 나오는 문장은 아닌 새로운 HTTP 메서드에 대해 이야기해 보겠습니다. [음악] 쿼리라고 해요. 이는 2010년 패치가 나온 이후 처음으로 추가된 새로운 HTTP 메서드입니다. [음악] Query는 요청 본문이 필요한 읽기 전용 요청을 위해 만들어졌습니다 . 간단한 필터링의 경우 쿼리 매개변수를 사용하는 GET 요청이 여전히 잘 작동하지만, 요청이 더 복잡해지거나 중첩 구조를 갖거나 페이로드가 커지면 URL 관리가 어려워집니다 . GET 요청에 본문을 포함하는 것은 브라우저, 클라이언트, 프록시, 서버 등 여러 요소에 따라 지원 여부가 다르기 때문에 항상 신뢰할 수 있는 것은 아닙니다. POST 요청도 본문을 허용하지만, 읽기 전용 작업에는 적합하지 않은 의미를 전달하여 여러 가지 문제를 야기할 수 있습니다. 이 쿼리는 안전성을 유지 하면서 복잡한 읽기 전용 작업을 지원하도록 설계되었습니다 . 아직 보편적인 지지는 아니지만 , 점차 확산되고 있습니다. 오늘은 여기까지입니다 . 이번 주에 다룬 주제에 대한 여러분의 생각을 댓글로 알려주세요. 이번 에피소드가 마음에 드셨다면 좋아요를 눌러 알고리즘의 신들에게 감사를 표하고, GitHub의 YouTube 채널을 구독하여 여러분의 모든 덕후스러운 콘텐츠를 즐겨보세요. 즐거운 여름 보내시고 다음 주에 뵙겠습니다. 안녕.
